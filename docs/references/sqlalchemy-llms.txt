# SQLAlchemy 2.0 Reference for II-Agent Development

> Key patterns and links for working with SQLAlchemy in this codebase.
> All models use SQLAlchemy 2.0 declarative style with async sessions.

## Core Patterns Used

- Declarative Mapping 2.0: https://docs.sqlalchemy.org/en/20/orm/declarative_styles.html
- Mapped Columns: https://docs.sqlalchemy.org/en/20/orm/mapped_attributes.html
- Relationships: https://docs.sqlalchemy.org/en/20/orm/relationships.html
- Async Session: https://docs.sqlalchemy.org/en/20/orm/extensions/asyncio.html
- Select: https://docs.sqlalchemy.org/en/20/orm/queryguide/select.html
- Insert/Update/Delete: https://docs.sqlalchemy.org/en/20/orm/queryguide/dml.html

## II-Agent Model Conventions

- Base class: ii_agent.core.db.base.Base (DeclarativeBase)
- Timestamps: TimestampColumn = TIMESTAMP(timezone=True)
- Mixin: TimestampMixin adds created_at/updated_at with server defaults
- Column style: Mapped[T] = mapped_column(...)
- Relationships: Mapped["OtherModel"] = relationship(back_populates="...")
- Indexes: defined via __table_args__ or Index()
- Soft delete: deleted_at DateTime column (nullable)
- Optimistic locking: version BigInteger column

## Async Patterns

- Engine: create_async_engine with asyncpg
- Session: async_sessionmaker[AsyncSession] with expire_on_commit=False
- Context manager: async with get_db_session_local() as db
- Pool: pool_size, max_overflow, pool_pre_ping=True, pool_recycle=3600
- Transactions: db.begin_nested() for SAVEPOINT
- Locking: .with_for_update() for SELECT...FOR UPDATE
